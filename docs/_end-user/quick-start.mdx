Get **Astrocyte** running — pick the deployment model that fits your stack and follow the commands.

## Choose your deployment model

<Tabs syncKey="deploy-mode">
<TabItem label="Library (in-process)">

Embed Astrocyte directly in your Python application. No separate server — memory operations run in-process.

**Best for:** prototyping, single-process agents, notebooks, custom orchestrators.

```bash
pip install -e astrocyte-py/
```

```python
from astrocyte import Astrocyte

brain = Astrocyte.from_config("astrocyte.yaml")

await brain.retain("Calvin prefers dark mode.", bank_id="user-123")
hits = await brain.recall("What theme does Calvin prefer?", bank_id="user-123")
```

The library alone does **not** start an HTTP server. See the [architecture overview](/design/architecture/) and [provider SPI](/plugins/provider-spi/) for wiring storage backends and LLM providers.

</TabItem>
<TabItem label="Standalone Gateway">

Run Astrocyte as a **REST API** (`astrocyte-gateway-py`) with Docker Compose. Agents call `/v1/retain`, `/v1/recall`, `/v1/reflect` over HTTP.

**Best for:** multi-agent deployments, microservice architectures, teams that want a shared memory service.

<Tabs syncKey="gateway-setup">
<TabItem label="Pre-built images">

No repo clone needed. Pull the pre-built images from GHCR and start with a single compose file.

```bash
docker pull ghcr.io/astrocyteai/astrocyte/astrocyte-postgres:latest
docker pull ghcr.io/astrocyteai/astrocyte/astrocyte-gateway-py:latest
```

Save this as `docker-compose.yml` and add your API key:

```yaml
services:
  postgres:
    image: ghcr.io/astrocyteai/astrocyte/astrocyte-postgres:latest
    environment:
      POSTGRES_USER: astrocyte
      POSTGRES_PASSWORD: astrocyte
      POSTGRES_DB: astrocyte
    ports:
      - "5433:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U astrocyte -d astrocyte"]
      interval: 3s
      retries: 5

  gateway:
    image: ghcr.io/astrocyteai/astrocyte/astrocyte-gateway-py:latest
    environment:
      DATABASE_URL: postgresql://astrocyte:astrocyte@postgres:5432/astrocyte
      ASTROCYTE_VECTOR_STORE: pgvector
      ASTROCYTE_GRAPH_STORE: age
      OPENAI_API_KEY: ${OPENAI_API_KEY}
    ports:
      - "8080:8080"
    depends_on:
      postgres:
        condition: service_healthy
```

```bash
OPENAI_API_KEY=sk-... docker compose up
```

</TabItem>
<TabItem label="From source">

Clone the repo and build locally — picks up any local changes.

```bash
cd astrocyte-services-py
cp .env.example .env        # edit ports / secrets as needed
docker compose up --build
```

From the repo root (alternative):

```bash
docker compose -f astrocyte-services-py/docker-compose.yml --env-file astrocyte-services-py/.env up --build
```

For production-shaped migrations and HNSW indexes:

```bash
./astrocyte-services-py/scripts/runbook-up.sh
```

</TabItem>
</Tabs>

Verify it's running:

```bash
curl http://127.0.0.1:8080/health   # health check (includes DB)
curl http://127.0.0.1:8080/docs     # OpenAPI explorer
```

Call the API:

```bash
# Retain a memory
curl -X POST http://127.0.0.1:8080/v1/retain \
  -H "Content-Type: application/json" \
  -d '{"content": "Calvin prefers dark mode.", "bank_id": "user-123"}'

# Recall memories
curl -X POST http://127.0.0.1:8080/v1/recall \
  -H "Content-Type: application/json" \
  -d '{"query": "What theme does Calvin prefer?", "bank_id": "user-123"}'
```

See [Production-grade HTTP service](production-grade-http-service/) for auth, TLS, rate limits, and observability.

</TabItem>
<TabItem label="Gateway Plugin">

Add long-term memory to LLM calls routed through an **existing API gateway** — zero agent code changes. Each plugin intercepts OpenAI-compatible `/chat/completions` requests and transparently calls Astrocyte for recall (pre-hook) and retain (post-hook).

**Best for:** organizations already routing LLM traffic through Kong, APISIX, or Azure APIM.

**Requires:** a running [standalone gateway](#) (see the Standalone Gateway tab) that the plugin calls via HTTP.

<Tabs syncKey="gateway-plugin">
<TabItem label="Kong">

```bash
# Copy plugin files to your Kong plugins directory
cp gateway-plugins/kong/handler.lua /path/to/kong/plugins/astrocyte/
cp gateway-plugins/kong/schema.lua  /path/to/kong/plugins/astrocyte/

# Enable via Admin API
curl -X POST http://localhost:8001/services/my-llm/plugins \
  -d "name=astrocyte" \
  -d "config.astrocyte_url=http://astrocyte-gateway:8900" \
  -d "config.default_bank_id=shared"
```

See [`gateway-plugins/kong/README.md`](https://github.com/AstrocyteAI/astrocyte/blob/main/gateway-plugins/kong/README.md) for declarative config and full options.

</TabItem>
<TabItem label="APISIX">

```bash
# Copy plugin to APISIX plugins directory
cp gateway-plugins/apisix/astrocyte.lua /path/to/apisix/plugins/

# Enable via Admin API
curl -X PUT http://127.0.0.1:9180/apisix/admin/routes/1 \
  -H "X-API-KEY: $APISIX_ADMIN_KEY" \
  -d '{
    "uri": "/v1/chat/completions",
    "upstream": { "nodes": { "llm-api:443": 1 }, "type": "roundrobin" },
    "plugins": {
      "astrocyte": {
        "astrocyte_url": "http://astrocyte-gateway:8900",
        "default_bank_id": "shared"
      }
    }
  }'
```

See [`gateway-plugins/apisix/README.md`](https://github.com/AstrocyteAI/astrocyte/blob/main/gateway-plugins/apisix/README.md) for declarative YAML config.

</TabItem>
<TabItem label="Azure APIM">

```bash
cd gateway-plugins/azure-apim

# Interactive — prompts for deployment method and config
./deploy.sh

# Or pass flags directly (Bicep example)
./deploy.sh --method bicep \
  --apim-name my-apim \
  --resource-group my-rg \
  --astrocyte-url https://astrocyte.internal:8900 \
  --api-key sk-...
```

Supports **Bicep**, **Terraform**, **APIOps**, and **portal** (manual paste). After deployment, add the fragments to your API policy:

```xml
<policies>
  <inbound>
    <base />
    <include-fragment fragment-id="astrocyte-recall-inject" />
  </inbound>
  <outbound>
    <base />
    <include-fragment fragment-id="astrocyte-retain-extract" />
  </outbound>
</policies>
```

See [`gateway-plugins/azure-apim/README.md`](https://github.com/AstrocyteAI/astrocyte/blob/main/gateway-plugins/azure-apim/README.md) for all deployment methods.

</TabItem>
</Tabs>

**Limitations:** Gateway plugins are request-scoped. They support recall and retain but **not** reflect, event streams, or poll connectors. See [Gateway edge & API gateways](gateway-edge-and-api-gateways/) for the full comparison.

## Prerequisites

- **Python 3.11+** (library and standalone gateway)
- **Docker** and **Docker Compose** (recommended for standalone gateway)
- **Kong**, **APISIX**, or **Azure APIM** (gateway plugin mode only)

## Wire an AI agent

Astrocyte does **not** run the agent loop (tools, checkpoints, multi-step graphs). Your framework does. Integrations map each framework's memory hooks to `retain` / `recall` / `reflect` through the same policy layer — see **[agent framework middleware](/plugins/agent-framework-middleware/)** for the full matrix.

<Tabs syncKey="agent-framework">
<TabItem label="LangGraph">

```python
from astrocyte import Astrocyte
from astrocyte.integrations.langgraph import AstrocyteMemory

brain = Astrocyte.from_config("astrocyte.yaml")
memory = AstrocyteMemory(brain, bank_id="user-123")

await memory.save_context(
    inputs={"question": "Theme preference?"},
    outputs={"answer": "Calvin prefers dark mode."},
    thread_id="thread-abc",
)
vars_for_prompt = await memory.load_memory_variables({})
```

```python
# Run the Astrocyte MCP server — agents connect via MCP protocol
from astrocyte import Astrocyte

from astrocyte.mcp import create_mcp_server

config = load_config("astrocyte.yaml")
```

```bash
astrocyte-mcp --config astrocyte.yaml
```

Exposes `memory_retain`, `memory_recall`, `memory_reflect`, and optionally `memory_forget`, graph, compile, and admin tools depending on `mcp:` configuration.

## Declarative routing with MIP

The **Memory Intent Protocol (MIP)** lets you declare routing rules so Astrocyte decides which bank, tags, and policies to apply — without routing logic in your application code.

```yaml
# mip.yaml — route student answers to per-student banks
rules:
  - name: student-answer
    match:
      all:
        - content_type: student_answer
        - metadata.student_id: present
    action:
      bank: "student-{metadata.student_id}"
      tags: ["{metadata.topic}"]
```

```python
brain = Astrocyte.from_config("astrocyte.yaml")

# MIP routes this to bank "student-stu-42" with tag "algebra"
await brain.retain(
    "2x + 3 = 7, so x = 2",
    bank_id="default",  # MIP overrides this
    metadata={"student_id": "stu-42", "topic": "algebra"},
    content_type="student_answer",
)
```

Mechanical rules resolve deterministically with zero LLM cost. When no rule matches, an optional intent layer can ask an LLM to decide. See the [MIP developer guide](/plugins/mip-developer-guide/) for the full match DSL, PII lockdown rules, compliance profiles, and testing recipes.
