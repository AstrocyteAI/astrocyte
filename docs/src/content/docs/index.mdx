    <TabItem label="MIP routing">
      <p class="quickstart-note">
        <strong>MIP</strong> (<a href="design/memory-intent-protocol/">Memory Intent Protocol</a>) lets you declare routing rules in <code>mip.yaml</code> and reference it from{" "}
        <code>astrocyte.yaml</code> with <code>mip_config_path: ./mip.yaml</code>. Astrocyte routes automatically —
        no routing code in your app.
      </p>

      <Tabs syncKey="mip-files">
        <TabItem label="mip.yaml">
```yaml
version: "1.0"
banks:
  - id: "student-{student_id}"
    description: Per-student academic memory
    compliance: pdpa
rules:
  - name: pii-lockdown
    priority: 1
    override: true          # compliance — intent layer cannot override
    match:
      pii_detected: true
    action:
      bank: private-encrypted
      tags: [pii, compliance]
      retain_policy: redact_before_store
  - name: student-answer
    priority: 10
    match:
      all:
        - content_type: student_answer
        - metadata.student_id: present
    action:
      bank: "student-{metadata.student_id}"
      tags: ["{metadata.topic}"]
```
        </TabItem>
        <TabItem label="astrocyte.yaml">
```yaml
profile: personal

provider_tier: storage
vector_store: postgres
vector_store_config:
  dsn: postgresql://localhost:5432/memories

llm_provider: openai
llm_provider_config:
  api_key: ${OPENAI_API_KEY}
  model: gpt-4o-mini

# PII detection — country-specific + LLM fallback
compliance_profile: pdpa
barriers:
  pii:
    mode: rules_then_llm
    countries: [SG, IN]

# Point to MIP routing policy
mip_config_path: ./mip.yaml

lifecycle:
  enabled: true
  ttl:
    archive_after_days: 90
    delete_after_days: 365
```
        </TabItem>
        <TabItem label="Python">
```python
from astrocyte import Astrocyte

brain = Astrocyte.from_config("astrocyte.yaml")

# MIP routes to bank "student-stu-42" with tag "algebra"
await brain.retain(
    "The quadratic formula solves second degree equations",
    bank_id="default",  # MIP overrides this
    metadata={"student_id": "stu-42", "topic": "algebra"},
    content_type="student_answer",
)

hits = await brain.recall("quadratic formula", bank_id="student-stu-42")
```

<span class="section-label" aria-hidden="true">⟩ What it does</span>
<h2 class="section-heading">Three operations, any backend</h2>
<div class="section-subheading">
  Retain, recall, and reflect — with pipelines and policies that stay valid when you swap storage or engines.
</div>

<div class="feature-grid not-content feature-grid--cols-3">
  <div class="feature-card">
    <h3>Retain</h3>
    <p>Store memories with automatic chunking, entity extraction, and embedding. PII scanning and content validation happen before anything reaches your backend.</p>
  </div>
  <div class="feature-card">
    <h3>Recall</h3>
    <p>Multi-strategy retrieval — semantic, graph, keyword, temporal — fused with reciprocal rank fusion and reranked. Token budgets enforced automatically.</p>
  </div>
  <div class="feature-card">
    <h3>Reflect</h3>
    <p>Synthesize answers from memory. Native engine reflect or automatic fallback via LLM. Disposition-aware personality for support, coding, or research agents.</p>
  </div>
</div>

<span class="section-label" aria-hidden="true">⟩ Governance</span>
<h2 class="section-heading">Built-in governance</h2>
<div class="section-subheading">
  Neuroscience-inspired policies that protect every operation — regardless of backend.
</div>

<div class="feature-grid not-content feature-grid--cols-3">
  <div class="feature-card">
    <h3>PII barriers</h3>
    <p>Regex, NER, or LLM-based scanning with country-specific patterns (SG, IN, UK, EU, AU, CA, JP, CN). Per-type actions, DLP output scanning, and compliance profiles (GDPR, HIPAA, PDPA).</p>
  </div>
  <div class="feature-card">
    <h3>Rate limits & budgets</h3>
    <p>Token bucket rate limiting, daily quotas, and per-operation token budgets. Prevent runaway agents.</p>
  </div>
  <div class="feature-card">
    <h3>Circuit breakers</h3>
    <p>Automatic degraded mode when backends go down. Empty recall, error, or cache — with controlled recovery.</p>
  </div>
  <div class="feature-card">
    <h3>Observability</h3>
    <p>OpenTelemetry spans and Prometheus metrics on every operation. Switch providers without rebuilding dashboards.</p>
  </div>
  <div class="feature-card">
    <h3>Access control</h3>
    <p>Per-bank read/write/forget/admin permissions. Principals, grants, and audit trails built in.</p>
  </div>
  <div class="feature-card">
    <h3>Data governance</h3>
    <p>Classification levels, compliance profiles (GDPR, HIPAA, PDPA), data residency, encryption, and DLP.</p>
  </div>
</div>

<span class="section-label" aria-hidden="true">⟩ Integrations</span>
<h2 class="section-heading">Agents, frameworks, models</h2>
<div class="section-subheading">
  One framework, every agent stack.
</div>

<CardGrid>
  <Card title="MCP Server" icon="laptop">
    Any MCP-capable agent (Claude Code, Cursor, Windsurf) gets memory via `astrocyte-mcp`. Zero code integration.
  </Card>
  <Card title="Memory Intent Protocol (MIP)" icon="document">
    Declarative memory routing — mechanical rules resolve deterministically, LLM judgment only when needed. One `mip.yaml` governs bank routing, compliance enforcement, and escalation across all agents.
  </Card>
  <Card title="Agent Frameworks" icon="puzzle">
    LangGraph, CrewAI, Pydantic AI, OpenAI Agents SDK, AG2 — thin middleware wires Astrocyte into each framework's memory abstraction.
  </Card>
  <Card title="LLM Providers" icon="translate">
    Multi-provider gateways (e.g. LiteLLM, Portkey, OpenRouter, Vercel AI Gateway) or direct SDKs — OpenAI, Anthropic, Bedrock, Azure, Vertex, Ollama — or bring your own via the LLM SPI.
  </Card>
  <Card title="Memory Portability" icon="right-arrow">
    Export and import memories between providers using Astrocyte Memory Archive (AMA), a portable newline-delimited JSON (JSONL) format. No vendor lock-in.
  </Card>
</CardGrid>
